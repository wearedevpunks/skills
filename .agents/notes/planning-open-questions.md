# Planning Open Questions

Spec and plan skills should not use `Open Questions` / unresolved questions as a silent escape hatch. For create-spec, meaningful user-answerable unknowns should route through a bounded `requirements-grill` phase; accepted grill outcomes that change scope should automatically run `write-backlog` before the spec is finalized. Tiny clarifications can be asked inline. Keep only explicitly deferred, externally blocked, or non-blocking items. This preserves review trust: open items explain real uncertainty, not skipped clarification.

For create-plan, scoped `Primary skills here` entries are planning inputs, not only executor labels. Load/use the relevant skill guidance before finalizing task boundaries, validations, RED targets, and review modes.

# Approval Phase

Approval is per user-visible scope unit.

## Steps

1. Present artifact candidates with scope, evidence, and constraints.
2. Record explicit user approval, rejection, or requested changes.
3. For each approved artifact set, write the artifact contract.
4. If assets are not durable, use `repo-asset-management` or record the upload blocker.
5. Route rejected or changed scope back to prototype or grill.

## Approved Artifact Contract

Each approved artifact set must include:

- scope unit
- artifact intent
- artifact links or files
- durable visual asset links when available
- evidence summary
- accepted constraints
- implementation notes
- acceptance checks
- staleness risks

## Rules

- No approved artifact set means no backlog conversion.
- Approval cannot be inferred from artifact existence.
- One approved artifact set belongs to one user-visible scope unit.

## Output

- Approved, rejected, and changed artifact sets.
- Artifact contracts.
- Phase handoff with next route: backlog when at least one approved artifact set exists.
